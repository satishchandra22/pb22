1. What is a Temporal Dead Zone?

Temporal dead zone is a block where let and const variables are stored until they get intialized with a value. We can not access the variables in temporal dead zone. if we try to access them , we get reference error. one way to solve this problem is to move let and const declarations to the top of the block.

3. Explain Local Scope, Block Scope, Functional Scope and Scope Chain in javascript

Local scope and functional scope are similar. all the varables declared inside a function and function parameters will have local scope, which meanse they can be accessed only inside the function. variables declared inside a block can only be accessed inside that block which is called block scope. When a variable is used in JavaScript, the JavaScript engine will try to find the variable's value in the current scope. If it could not find the variable, it will look into the outer scope and will continue to do so until it finds the variable or reaches global scope. this is called scope chain.

7. Explain passed by value and passed by reference

when ever we assign primitive datatypes to a new variable, it always creates a new copy of the value. All the primitive datatypes are passed by the value. on otherhand, nonprimitive datatypes or objects are passed by reference. whenevr you assign an object to a new variable, reference to that object is stored in that varible.
